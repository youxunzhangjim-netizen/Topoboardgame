Clifford Reversi: Standard Pauli Set

Algebra set: Clifford Reversi is one game with configurable algebra and opening rules. Standard Pauli keeps the ordinary four-stone Reversi opening. Choose Physical Stabilizer to use physical qubit sites, alternate initial rules, measurements, phases, and ancillas.

3D boards: R3 uses a rotatable cubic graph, T2 uses a solid torus, and S2 uses a latitude-ring sphere. Click graph vertices to place stones; drag to rotate and wheel to zoom. The same topology-aware Clifford rules remain active on these surfaces.

Move rule: place a Pauli-labelled stone on an empty vertex only when it brackets at least one opponent chain along a topology-aware ray. Rays use the selected topology, so 2D RBC, torus, Klein bottle, RP2, S2 latitude, and 4D grid boards use their graph normalization instead of flat boundary assumptions.

Flip rule: every bracketed opponent stone changes to the acting color. Its Pauli label is transformed by the selected Clifford generator, and twisted seams apply their transport while the ray is resolved.

Pauli math: I=(0,0), X=(1,0), Z=(0,1), Y=(1,1). The symplectic product is x1*z2 + z1*x2 mod 2. Labels anticommute when this value is 1.

H and S: without phase signs, H swaps X and Z and leaves Y as Y; S maps X to Y, Y to X, and Z to Z. With Phase Signs on, H maps Y to -Y and S maps Y to -X, so flipped stones display +X, -Y, etc.

Dynamics choices: Pauli noise changes labels with seeded random rolls. Basic Floquet ages pieces and diffuses fields. Clifford Floquet applies H/S at marked defect vertices. Virasoro Floquet evolves a stress field T(v): phase 1 diffuses stress, phase 2 applies L0 scaling near marked vertices, phase 3 focuses stress like L1, and optional anomaly phases add L2-style stress.

Clifford Reversi: Physical Stabilizer Set

Algebra set: this is the Physical Stabilizer configuration of the same Clifford Reversi game. Switch Algebra Set back to Standard Pauli Reversi to restore the ordinary opening and controls.

Physical sites: empty vertices are identity I with no active excitation. Occupied vertices store owner, Pauli I/X/Y/Z, sign, phase modulo four, and optional ancilla data. Black is the positive sector and white is the negative sector.

Initial states: Stabilizer Vacuum is empty with positive checks. Sparse Errors seeds low-density X/Y/Z faults. Paired Defects creates nearby opposite-sign pairs. Domain Wall Seed separates positive and negative domains. Prepared Circuit applies ancilla preparations, H, S, CNOT, CZ, and measurement to a vacuum start.

Reversi action: placement still requires a normal topology-aware bracket. Each flipped site changes owner and sign, then receives the selected H or S transform plus any seam transport. Pauli phase is tracked modulo four.

Ancillas: prepare Z0, Z1, X+, X-, or magic ancillas on empty sites. Entangle an occupied control and target with CNOT or CZ when at least one endpoint is an ancilla. Ancillas may be measured in X/Z or discarded.

Phase actions: S, S dagger, and Z are Clifford actions. Phase Kick accepts a numeric angle and marks non-quarter-turn use as a non-stabilizer approximation.

Stabilizer Recovery problem: this optional physical problem treats local Clifford/Reversi actions as Pauli-frame recovery. It adds graph-neighborhood X/Z syndromes, topology-cycle logical checks, recovery time, seeded measurement errors, and a dedicated QEC export without changing ordinary Reversi.

Measurements: measure a local Pauli, connected-domain parity, bracketed-line parity, neighborhood stabilizer, or logical-cycle parity. Seeded measurement errors use the configured error rate, and the measured site collapses to the selected X or Z basis.

Observables: every action records Pauli/sign/phase distributions, ancilla count, syndrome weight, local X/Z violations, global parity, commutation conflicts, logical sector, recovery state, measurement errors, and non-Clifford resource use.

Physical Virasoro/CFT Reversi Rules

Approximation: this is a discrete graph simulator inspired by CFT. Correlations, conformal blocks, stress, entropy, and topology sectors are lattice estimators rather than exact continuum observables.

Fields: an empty vertex is identity. Occupied vertices carry a domain sign and a primary field. The default Ising CFT uses c=1/2, identity h=0, sigma h=1/16, and epsilon h=1/2.

Initial states: Vacuum is empty. Domain Wall Seed creates opposite spin domains. Four Sigma Block tracks identity and epsilon blocks. Boundary Condition Change places seam/boundary insertions. Thermal Sample uses seeded temperature-weighted sigma/epsilon insertions.

Reversi interval: a placement must bracket an opponent path. The path is treated as a discrete conformal interval: signs flip and Ising OPE rules update fields and channels. Sigma x sigma has identity or epsilon channels, sigma x epsilon gives sigma, and epsilon x epsilon gives identity.

Virasoro actions: N=1 enables L_-1, L_0, L_1. N=2 also enables L_-2 splitting and L_2 concentration. Overlap of N=2 actions can create central-charge anomaly markers.

Measurements: measure line parity, hidden OPE channels, the four-point block, region entropy, or local/region stress. Measurement errors are seeded and logged.

Output: every turn records primary/sign counts, domain walls, connected domains, correlation estimates, cross-ratio, block weights, stress T(v), entropy, mutual information, OPE transitions, anomaly events, and topology sector.

Anyon Jump Chess Rules

3D boards: R3, T2, and S2 use rotatable graph embeddings. Legal hops, jumps, braid paths, fusion sites, labels, and braid-state shadows are drawn directly on the 3D board.

Move rule: a token may hop to an adjacent empty vertex or jump over one occupied neighboring token into the next empty vertex in the same topology-aware direction. The move stores the visited path as a worldline segment.

2D RBC: when selected, each boundary exit is linked to one fixed random boundary square for the current game. Anyon hops and jumps use those links through the same graph-neighbor system as the other topologies.

Symbols: toric code labels are 1, e, m, and ψ. Ising labels are 1, σ, and ψ. Fibonacci labels are 1 and τ. Board tokens use these Greek symbols directly.

Vacuum 1: 1 is the trivial topological charge, meaning no nontrivial anyon remains. It acts like the identity in fusion: 1 x a = a.

Braiding: a jump over a mutually nontrivial target can append a braid generator σi or σi-1. Abelian parity toggles a Z2 braid state for e around m or m around e. Word Exact stores the full noncommutative braid word. Non-Abelian Channel also updates a hidden symbolic fusion channel.

Unbraiding: select a braided token. The valid target is marked with a green UNBRAID badge. Clicking that target applies the next inverse generator. Full unbraiding requires the reverse inverse sequence; a wrong target or wrong sign appends a new braid instead of clearing the word. A successful inverse also reduces the visible braid count.

General Z_n phase: the selectable non-vacuum charges are α1 through αn-1. Fusion adds grades modulo n, so αi x αj = α(i+j) mod n, with grade 0 equal to vacuum 1. A positive braid adds +1/n, an inverse unbraid subtracts 1/n, and the token shows the signed net phase as a flat shadow under the piece.

Toric code fusion: e x e = 1, m x m = 1, ψ x ψ = 1, e x m = ψ, e x ψ = m, and m x ψ = e. Fusion to 1 is vacuum fusion and removes or neutralizes the fused pair according to config.

Toric Memory Unbraid: the physical-problem option initializes local e and m pairs with vacuum total charge, records the logical winding sector, and shows QEC observables after every move. Crossing a noncontractible cycle can create a logical error; exact inverse braids reduce braid memory; complete vacuum fusion is recorded as vacuum recovery.

QEC export: JSON includes vertex-based initial state, move history, braid and fusion histories, logical-sector history, memory lifetime, logical-error rate, unbraid success rate, and the final physical answer.

Excitation Energy: both players start with 12 energy. The selector offers every non-vacuum type in the chosen model: e, m, ψ for Toric Code; σ, ψ for Ising; τ for Fibonacci; and every αk grade for Z_n. Exciting one on an empty vertex consumes one turn and its model cost. Dropping an owned anyon or completing a vacuum fusion returns its gap energy after the configured loss rate.

Non-Abelian fusion: Ising uses σ x σ = 1 + ψ. The plus sign lists alternative channels, not simultaneous products: resolving to 1 removes both incoming σ anyons as vacuum, while resolving to ψ replaces the pair with one ψ excitation. Also, σ x ψ = σ and ψ x ψ = 1. Fibonacci similarly uses τ x τ = 1 + τ. Measurements can reveal hidden channels when measurement rules are enabled.

Entanglement distance: Infinite keeps fusion-channel memory regardless of graph separation. Finite Distance measures shortest legal graph steps and decoheres that hidden pair channel when the anyons separate beyond the selected distance. Braid words remain recorded independently.

Dynamics choices: Anyon noise can create pairs or flip e/m on configured noisy places. Anyon Floquet evaluates braid histories, activates fusion sites, moves twist defects, and can apply seam automorphisms such as e<->m. Virasoro Floquet is available as a shared stress-field experiment on the underlying graph.

Virasoro Go Rules

3D boards: R3 uses a cubic graph, T2 uses the complete torus surface, and S2 uses a latitude-ring sphere. Stones, legal vertices, conformal weights h, stress rings, and Virasoro previews are rendered on the interactive board.

Base Go: this mode uses ordinary graph-Go rules. Stones live on graph vertices. Connected groups, liberties, captures, suicide rejection, superko, two-pass counting, and area scoring all use graph adjacency from the selected topology.

Lattices: square uses four graph neighbors, honeycomb uses three, and triangular uses six: ±x, ±y, (+x,-y), and (-x,+y). Captures require enclosing every remaining graph liberty, including diagonal liberties exposed at a topology boundary.

2D RBC: the 2D RBC topology creates one static random map from every boundary exit to another boundary point. Liberties, groups, captures, stress movement, and scoring all use that graph.

Optional Virasoro layer: every vertex has a stress value T(v), initially 0. Each connected stone group has conformal weight h = group_size / 2. The central charge c defaults to 1. With max mode N = 1, the enabled actions are L_-1, L_0, and L_1. With N = 2, L_-2 and L_2 are also available.

Turn choice: a player may place a normal Go stone, pass, agree to count after two passes, or spend the turn on one Virasoro action when the layer is on. Normal Go captures still happen only by zero liberties.

L_-1: choose a friendly connected group and a graph direction. For each empty liberty adjacent to the group, move up to one unit of stress from that liberty to the next empty vertex along that direction. L_-2 does the same shift over two graph steps.

L_0: choose a friendly connected group. Add +1 stress to every empty liberty adjacent to that group. The stress is marked as controlled by the acting player.

L_1: choose an empty target vertex. Move up to one unit of stress from each empty neighboring vertex into the target. L_2 pulls from distance-two axis neighbors.

Stress pressure: for a group, enemyStressPressure is the sum of T(v) over empty liberties controlled by the opponent. If enemyStressPressure > h + c, the group is marked unstable. By default this is a warning marker only; the optional Remove Group setting enables experimental removal.
