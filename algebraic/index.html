Clifford Reversi Rules

3D boards: R3 uses a rotatable cubic graph, T2 uses a solid torus, and S2 uses a latitude-ring sphere. Click graph vertices to place stones; drag to rotate and wheel to zoom. The same topology-aware Clifford rules remain active on these surfaces.

Move rule: place a Pauli-labelled stone on an empty vertex only when it brackets at least one opponent chain along a topology-aware ray. Rays use the selected topology, so 2D RBC, torus, Klein bottle, RP2, S2 latitude, and 4D grid boards use their graph normalization instead of flat boundary assumptions.

Flip rule: every bracketed opponent stone changes to the acting color. Its Pauli label is transformed by the selected Clifford generator, and twisted seams apply their transport while the ray is resolved.

Pauli math: I=(0,0), X=(1,0), Z=(0,1), Y=(1,1). The symplectic product is x1*z2 + z1*x2 mod 2. Labels anticommute when this value is 1.

H and S: without phase signs, H swaps X and Z and leaves Y as Y; S maps X to Y, Y to X, and Z to Z. With Phase Signs on, H maps Y to -Y and S maps Y to -X, so flipped stones display +X, -Y, etc.

Dynamics choices: Pauli noise changes labels with seeded random rolls. Basic Floquet ages pieces and diffuses fields. Clifford Floquet applies H/S at marked defect vertices. Virasoro Floquet evolves a stress field T(v): phase 1 diffuses stress, phase 2 applies L0 scaling near marked vertices, phase 3 focuses stress like L1, and optional anomaly phases add L2-style stress.

Anyon Jump Chess Rules

3D boards: R3, T2, and S2 use rotatable graph embeddings. Legal hops, jumps, braid paths, fusion sites, labels, and braid-state shadows are drawn directly on the 3D board.

Move rule: a token may hop to an adjacent empty vertex or jump over one occupied neighboring token into the next empty vertex in the same topology-aware direction. The move stores the visited path as a worldline segment.

2D RBC: when selected, each boundary exit is linked to one fixed random boundary square for the current game. Anyon hops and jumps use those links through the same graph-neighbor system as the other topologies.

Symbols: toric code labels are 1, e, m, and ψ. Ising labels are 1, σ, and ψ. Fibonacci labels are 1 and τ. Board tokens use these Greek symbols directly.

Vacuum 1: 1 is the trivial topological charge, meaning no nontrivial anyon remains. It acts like the identity in fusion: 1 x a = a.

Braiding: a jump over a mutually nontrivial target can append a braid generator σi or σi-1. Abelian parity toggles a Z2 braid state for e around m or m around e. Word Exact stores the full noncommutative braid word. Non-Abelian Channel also updates a hidden symbolic fusion channel.

Unbraiding: select a braided token. The valid target is marked with a green UNBRAID badge. Clicking that target applies the next inverse generator. Full unbraiding requires the reverse inverse sequence; a wrong target or wrong sign appends a new braid instead of clearing the word. A successful inverse also reduces the visible braid count.

General Z_n phase: this option keeps toric-code token types but adds an observable braid phase k/n. A positive braid adds +1/n, an inverse unbraid subtracts 1/n, and the token shows the signed net phase as a flat shadow under the piece.

Toric code fusion: e x e = 1, m x m = 1, ψ x ψ = 1, e x m = ψ, e x ψ = m, and m x ψ = e. Fusion to 1 is vacuum fusion and removes or neutralizes the fused pair according to config.

Excitation Energy: both players start with 12 energy. The default toric gaps and excitation costs are Δe=2, Δm=2, and Δψ=4. Exciting e, m, or ψ on an empty vertex consumes one turn and its cost. Dropping an owned anyon or completing a vacuum fusion returns its gap energy after the configured loss rate.

Non-Abelian fusion: Ising uses σ x σ = 1 + ψ, σ x ψ = σ, and ψ x ψ = 1. Fibonacci uses τ x τ = 1 + τ. Measurements can reveal hidden channels when measurement rules are enabled.

Dynamics choices: Anyon noise can create pairs or flip e/m on configured noisy places. Anyon Floquet evaluates braid histories, activates fusion sites, moves twist defects, and can apply seam automorphisms such as e<->m. Virasoro Floquet is available as a shared stress-field experiment on the underlying graph.

Virasoro Go Rules

3D boards: R3 uses a cubic graph, T2 uses the complete torus surface, and S2 uses a latitude-ring sphere. Stones, legal vertices, conformal weights h, stress rings, and Virasoro previews are rendered on the interactive board.

Base Go: this mode uses ordinary graph-Go rules. Stones live on graph vertices. Connected groups, liberties, captures, suicide rejection, superko, two-pass counting, and area scoring all use graph adjacency from the selected topology.

Lattices: square uses four graph neighbors, honeycomb uses three, and triangular uses six: ±x, ±y, (+x,-y), and (-x,+y). Captures require enclosing every remaining graph liberty, including diagonal liberties exposed at a topology boundary.

2D RBC: the 2D RBC topology creates one static random map from every boundary exit to another boundary point. Liberties, groups, captures, stress movement, and scoring all use that graph.

Optional Virasoro layer: every vertex has a stress value T(v), initially 0. Each connected stone group has conformal weight h = group_size / 2. The central charge c defaults to 1. With max mode N = 1, the enabled actions are L_-1, L_0, and L_1. With N = 2, L_-2 and L_2 are also available.

Turn choice: a player may place a normal Go stone, pass, agree to count after two passes, or spend the turn on one Virasoro action when the layer is on. Normal Go captures still happen only by zero liberties.

L_-1: choose a friendly connected group and a graph direction. For each empty liberty adjacent to the group, move up to one unit of stress from that liberty to the next empty vertex along that direction. L_-2 does the same shift over two graph steps.

L_0: choose a friendly connected group. Add +1 stress to every empty liberty adjacent to that group. The stress is marked as controlled by the acting player.

L_1: choose an empty target vertex. Move up to one unit of stress from each empty neighboring vertex into the target. L_2 pulls from distance-two axis neighbors.

Stress pressure: for a group, enemyStressPressure is the sum of T(v) over empty liberties controlled by the opponent. If enemyStressPressure > h + c, the group is marked unstable. By default this is a warning marker only; the optional Remove Group setting enables experimental removal.
